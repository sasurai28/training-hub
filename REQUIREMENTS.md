# トレーニングハブ PWA 要件定義書

- 作成日: 2026-06-12
- 用途: この文書をリポジトリ直下に `REQUIREMENTS.md` として置き、Claude Code に「REQUIREMENTS.md の Phase 1 を実装して」と指示する
- 利用者: 1名（個人用）。早朝のジム・屋外でのスマホ片手操作が前提

---

## 1. 目的と背景

2つの目標を 1 本のアプリで管理する。

1. **マッチョ化**: 2026年9月中旬の旅行までに、上半身中心の筋肥大＋直前の仕上げ
2. **サブ3**: 2027年の大阪マラソン（2月想定）で 3 時間切り

期分け:

- **〜2026年9月中旬**: 筋トレ優先期。ジム週2、ランは週20〜30kmの維持モード、ピックルボール週2
- **2026年9月中旬〜**: ラン主軸期。段階的に週55〜70kmまでビルドし、マラソンへ

### 設計思想（実装上の最優先事項）

1. **記録は10秒以内**。入力が面倒なアプリは続かない
2. **開いた瞬間に「今日やること」が見える**。ジムで考えない
3. **続けたくなる仕掛け**: 前回超え✓、週次目標の達成リング、写真比較
4. **ローカル完結**: サーバー・認証・課金なし。個人データを外部送信しない

---

## 2. 技術スタック

- React + Vite + TypeScript
- PWA: manifest + Service Worker。**初回ロード後はオフライン完全動作**、ホーム画面追加対応
- ホスティング: GitHub Pages（GitHub Actions で main への push 時に自動デプロイ）
- データ保存: localStorage（構造化ログ）。写真のみ IndexedDB（Phase 2）
- バックアップ: JSON エクスポート / インポート
- 外部 API: Phase 3 まで一切使わない

## 3. フェーズ計画

### Phase 1 — MVP（最初の実装対象）

#### 1. ホーム（今日）
- 今日の日付と、週間スケジュールから判定した「**今日のメニュー**」を最上部に表示。ジム日なら種目リストまで展開
- イベントカウントダウン（直近イベントを大きく、残りは小さく）
- クイック記録ボタン: ジム / ラン / ピックルボール / 体重・タンパク質
- 今週の達成状況（目標に対する進捗バー: 例「ジム 1/2回」「ラン 12.5/25km」）

#### 2. 筋トレ記録
- 今日のメニューの種目リストを表示。種目タップでセット入力へ
- **前回実績がデフォルト値として自動入力済み**。重量は ±2.5kg、回数は ±1 のステッパー（直接数値入力も可）
- セット完了タップで即保存。前回の同セットを重量または回数で上回ったら ✓ と軽い演出（このアプリの一番気持ちいい瞬間にする）
- 種目の追加・削除・並べ替え、メニュー外種目のその場追加が可能

#### 3. ラン記録
- 距離 km（小数1桁）/ 時間（分:秒）/ 種別（ジョグ・ポイント練習・レース）/ メモ
- 保存時にペース（min/km）を自動計算して表示
- 週間走行距離を自動集計し、ホームとカレンダーに反映

#### 4. ピックルボール記録
- ワンタップ「やった」＋任意メモのみ（軽量でよい）

#### 5. 体重・タンパク質
- 体重: 数値入力（前回値 ±0.1 ステッパー）。最新値と前週平均との差を表示
- タンパク質: 「達成 / 未達」のワンタップのみ。グラム入力はさせない（目安は体重×1.6〜2g/日）

#### 6. カレンダー
- 月表示。活動種別ごとの色ドット（ジム / ラン / ピックルボール）
- 日タップで詳細の閲覧・編集。週ごとの走行距離合計も表示
- 評価軸は連続日数ストリークではなく「**週次目標の達成**」（毎日やる前提の設計は破綻するため）

#### 7. 設定
- 週間スケジュール編集（曜日 → 活動・メニューの割当）
- メニュー編集 / 目標編集 / イベント（日付）編集
- JSON エクスポート（ファイルDL）/ インポート
- ダーク・ライト切替

### Phase 2 — 可視化と振り返り（2026年6月末 目安）

- グラフ: 体重推移（7日移動平均線付き）/ 週間走行距離の棒グラフ / 主要種目の重量推移（種目選択式）
- 週次レビュー: 日曜に今週サマリー（ジム回数・走行距離・タンパク質達成日数・体重変化、前週比）
- 写真ログ: 週1撮影をホームのバッジで促す。IndexedDB 保存、同アングル撮影ガイド、任意2枚の並列比較ビュー
- バックアップリマインド: 最終エクスポートから2週間経過で表示

### Phase 3 — ラン強化期（2026年9月〜）

- **ラン計画エンジン**: マラソン当日から逆算した期分けテンプレート（土台 → ビルド → ピーク → テーパー）で週次メニュー（距離・ポイント練内容）を生成し、ホームの「今日のメニュー」に反映。実績に応じて手動調整可
- **負荷アラート**: 週間走行距離が前週比 +10% を超えたら警告（故障予防）
- **レース換算**: 直近のレース / タイムトライアルから VDOT 相当でマラソン予測タイムを表示。ハーフ 1:25〜1:27 を「サブ3圏内」の目安としてチェックポイント表示
- （任意）GPX / CSV インポート、Strava API 連携（トークンはローカル保存・明示同意）

## 5. データモデル（概要）

実装時に細部は調整してよいが、骨格は以下を想定。

```ts
type ActivityType = 'gym' | 'run' | 'pickleball';

interface GymSet { weightKg: number; reps: number; }

interface DayLog {
  date: string; // 'YYYY-MM-DD'
  gym?: { menuId: string; exercises: { name: string; sets: GymSet[] }[] };
  run?: { km: number; durationSec: number; kind: 'jog' | 'workout' | 'race'; memo?: string };
  pickleball?: { done: boolean; memo?: string };
  weightKg?: number;
  proteinOk?: boolean;
  photoId?: string; // Phase 2（IndexedDB のキー）
}

interface Menu {
  id: string;
  name: string; // 例: 'A日（プッシュ）'
  exercises: { name: string; targetSets: number; targetRepsMin: number; targetRepsMax: number }[];
}

interface Settings {
  weeklySchedule: Record<
    'mon' | 'tue' | 'wed' | 'thu' | 'fri' | 'sat' | 'sun',
    { type: ActivityType | 'rest'; menuId?: string }[]
  >;
  goals: { gymPerWeek: number; runKmPerWeek: number; proteinDaysPerWeek: number; targetWeightKg?: number };
  events: { name: string; date: string }[];
}
```

- localStorage キー: `th:logs` / `th:menus` / `th:settings`
- 書き込みは入力のたびに即時（保存ボタン待ちでデータを失わない）

## 6. 初期データ

### メニュー

- **B日（プル）**: ラットプルダウン / シーテッドロウ / アームカール / 腹筋（プランク60秒×3 または クランチ15×3）

### 週間スケジュール初期値（仮。設定画面で本人が調整する前提）
- 火: ジム（A/B 交互） / 金: ジム（A/B 交互）
- ラン: 週3回・合計20〜30km（〜9月中旬）
- ピックルボール: 週2回

### 目標初期値
- ジム 週2回 / ラン 週25km / タンパク質達成 週6日以上 / 体重目標は本人入力

### イベント初期値（日付は仮置き、設定で更新）
- 2026年7月中旬: ピックルボール市民大会
- 2026年9月中旬: プール付きコテージ旅行
- 2027年2月（日付確定後に更新）: 大阪マラソン

## 7. UI デザイン方針

- **ダークモードを既定**（早朝4時台のジムで眩しくないこと）。ライト切替可
- 装飾より速度。大きな数字、大きなタップ領域（44px 以上）、主要ボタンは親指リーチ内
- アクセントカラーは1色に絞る。アニメーションは「前回超え✓」など記録完了フィードバックに限定
- このアプリのシグネチャーは「前回超えの瞬間」。ここだけは明確に気持ちいい演出にする
- `prefers-reduced-motion` を尊重、キーボードフォーカス可視

---

## 8. 非機能要件

- iOS Safari / Android Chrome 対応。ホーム画面追加で全画面起動
- 初回ロード後はオフライン完全動作。リロード・再起動でデータが消えない
- 起動 → 今日のメニュー表示まで1秒以内
- 記録所要時間: ジム1種目 10秒以内 / ラン 15秒以内
- 個人データの外部送信なし

## 9. Phase 1 受け入れ基準

- [ ] オフラインで起動し、今日のメニューが即座に表示される
- [ ] 前回値がプリセットされた状態で、1種目3セットを各10秒以内で記録できる
- [ ] ラン・ピックルボール・体重・タンパク質が記録できる
- [ ] カレンダーに活動が色分け表示され、週次目標の進捗が見える
- [ ] JSON エクスポート → 別ブラウザでインポートし、完全に復元できる
- [ ] スマホのホーム画面に追加してアプリとして起動できる
